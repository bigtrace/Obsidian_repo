🔭 What is OpenClaw?
Think of it as a personal Bloomberg terminal that you can talk to — and that works for you overnight.
💡

The one-line pitch: OpenClaw is an always-on AI assistant you run on your own server. It connects to the messaging apps you already use (Telegram, WhatsApp, Slack…), can execute code, fetch data, and schedule automated tasks — all controlled by you, no data sent to a third-party SaaS platform.

📱 It lives in your chat

You message it on Telegram (or WhatsApp, Slack, etc.) like texting a very capable analyst. It reads your question, thinks, uses tools, and replies — no app to open.

Analogy: Like having an analyst who's always on call and replies on WhatsApp.

⏰ It works while you sleep

Cron jobs let you schedule tasks — fetch data, run scripts, generate reports — at any time. Every morning at 8 AM SGT, six research digests land in your Telegram automatically.

Analogy: Like a Bloomberg alert + morning briefing, but fully customisable.

🧠 It remembers context

Unlike a standard chatbot, OpenClaw has persistent memory files, a workspace, and skills. It learns your preferences and can maintain research threads across sessions.

Analogy: Like an analyst who actually reads last week's notes before your call.
🏗️ How It's Structured
Three layers: the Gateway (server), the Agent (brain), and the Workspace (memory + tools).
📱 You
Telegram / WhatsApp / Slack / Web — wherever you message
⟷
🌐 Gateway (Server Process)
The always-on daemon running on your server (VPS/Mac/Pi). Routes messages between your chat apps and the AI model. Manages channels, authentication, cron scheduler, and delivery queues. Config: ~/.openclaw/openclaw.json
↓
🤖 Agent (Claude / GPT)
The AI model (currently Claude Sonnet 4.6). Receives your message + workspace context, calls tools (web search, code execution, file read/write, memory), and crafts a reply. Governed by SOUL.md, AGENTS.md, and skills.
↓
📁 Workspace
The agent's "desk" — files it can read, write, and execute. Contains memory, scripts, digests, and skills. Path: ~/.openclaw/workspace/
🧠 Memory Files
MEMORY.md, daily notes — persistent context across sessions
🐍 Scripts
Python/bash scripts the agent can run — data fetchers, digest generators
🎯 Skills
Modular playbooks (SKILL.md files) that teach the agent domain-specific workflows
⏰ Cron Jobs
Scheduled tasks — runs scripts automatically, delivers reports to Telegram

🧬 SOUL.md

The agent's character document. Defines how it speaks, whether it has opinions, and what it prioritises. Like an onboarding doc for a new analyst.

📋 AGENTS.md

Operating manual — when to check email, how to handle group chats, memory management rules. The agent reads this on every session start.

🎯 skills/SKILL.md

Modular knowledge packs. Each skill teaches the agent a specific workflow — e.g. "how to generate an A-Share report" or "how to sync to Obsidian."

⏰ cron/jobs.json

The job scheduler. Each entry has a cron expression, a script to run, and a delivery target (Telegram chat ID). Runs independently of any active conversation.

🛠️ How to Extend It (No Coding Required for Simple Tasks)
The PM doesn't need to write code — just describe what you want.

💬 Just ask in Telegram

"Add a daily digest tracking semiconductor supply chain news."

The agent will: find RSS sources, write the Python script, schedule the cron job, and start delivering it tomorrow morning. No code written by you.

🎯 Add a new skill

Skills are markdown files that teach the agent how to do something specific. clawhub search "earnings transcript" to find pre-built skills from the community, or ask the agent to create a custom one.

📊 Connect your own data

Point a Python script at any API — Bloomberg Terminal API, FactSet, Refinitiv, your prime broker portal — and the agent can read, analyse, and summarise that data on a schedule.

🔗 Chain reports together

The TL;DR generator reads all 6 digests and synthesises a single master note. You can chain any reports: "if the AI digest mentions NVDA + disruption, add a flag to the morning watchlist."

⚡

The key insight for a PM: OpenClaw is not a fixed product — it's a programmable research assistant infrastructure. The 6 morning digests took about 4 hours to build from scratch in a single conversation. Every new data source, new theme, or new alert is 10-30 minutes of conversation away. The marginal cost of adding a new research workflow is near zero.

🔒 Security & Privacy
Your data stays on your server.

🏠 Self-hosted

Runs on your own VPS (AWS, DigitalOcean, or home server). No data goes to an OpenClaw cloud. Your research, positions, and questions never leave your infrastructure.

🤖 AI model calls

Your messages are sent to Anthropic (Claude) or OpenAI for inference — the same as using Claude.ai directly. Use enterprise agreements if needed for data residency.

📱 Channel security

Telegram/WhatsApp bots are auth-gated. Only whitelisted user IDs can interact with the agent. Group policies control who can trigger actions.

🔑 API credentials

Stored locally in ~/.openclaw/credentials/. Not transmitted anywhere except to the intended API provider (GitHub, Telegram, etc.).
